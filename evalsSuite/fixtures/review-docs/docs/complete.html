Telemetry Dashboard

The Telemetry Dashboard is a lightweight web UI for monitoring service health, latency, and error trends.

Purpose

Provide a single-pane view for SREs and developers to quickly triage incidents.

Features

  • Live charts for p50/p95 latency, request rate, and error counts.
  • Service-level filtering by environment (dev, staging, prod).
  • Export snapshots to CSV for offline analysis.

Getting Started

  1. Install dependencies with npm install.
  2. Set TELEMETRY_API_URL in .env.
  3. Run npm start and open http://localhost:3000.

Usage

Select a service, choose a time window, and hover on charts to see exact values.

Examples

// filter to prod services
npm start -- --env prod
// export last 24h as CSV
npm run export -- --range 24h --service api

Troubleshooting

  • If charts are empty, verify the API URL and network access.
  • If exports fail, ensure write access to the exports/ directory.
