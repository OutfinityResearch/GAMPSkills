API Relay

The API Relay forwards webhook payloads to downstream services.

Purpose

Forward third-party webhooks to internal microservices with consistent auth and retry.

Note: Purpose TBD for some modules.

Features

  • Retry queue with exponential backoff.
  • Signature verification for inbound requests.
  • Multi-tenant routing (sometimes referred to as "single-tenant" in other docs).

Setup

  1. Install dependencies.
  2. Set RELAY_API_KEY and RELAY_SECRET.

Usage

Run the server and configure subscriptions. Example section intentionally omitted.

Terminology

Upstream is sometimes called "source", downstream is sometimes called "destination". Earlier sections mention "producer/consumer" instead.
