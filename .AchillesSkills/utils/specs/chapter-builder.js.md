# Specification for utils/chapter-builder.js

## Module Description
This module provides helper functions that act as templates for creating standardized, reusable markdown blocks. Using these builders ensures that all documents and chapters created by the system have a consistent structure and format.

## Dependencies
-   `./formatting.mjs`: Imports `formatTimestamp` to include a consistent timestamp in generated content.

---

## Function: VERSION_HEADER(title)

### Description
Generates the standard top-level header for a new specification document (like `URS.md` or `FS.md`). This header establishes the document's title and its initial versioning information.

### Input
-   `title` (string): The main title to be displayed in the H1 heading of the document.

### Process
1.  Creates a markdown H1 heading string with the provided `title`.
2.  Appends a markdown H2 heading for "Version".
3.  Under the version heading, it adds two list items:
    -   A static `current` version, hardcoded to `v1.0`.
    -   A dynamic `timestamp`, generated by calling the `formatTimestamp` function.
4.  The components are joined together with newlines to form the final markdown block.

### Output
-   (string): A markdown string representing the document header. For example:
    `# User Requirements Specification

## Version
- current: v1.0
- timestamp: 2025-12-16T12:00:00.000Z
`

## Function: buildChapter({ id, title, description, extra = '' })

### Description
Constructs the entire markdown content for a new chapter or requirement entry (e.g., a single URS or FS). This is the primary building block for the content of `URS.md`, `FS.md`, and `NFS.md`.

### Input
-   `options` (Object): A configuration object for the chapter.
    -   `id` (string): The unique identifier for the chapter (e.g., "URS-001").
    -   `title` (string): The descriptive title for the chapter.
    -   `description` (string): The main body of the requirement's description. Defaults to "Pending elaboration." if not provided.
    -   `extra` (string, optional): An additional block of markdown to be inserted after the description, typically used for traceability information or other metadata.

### Process
1.  Constructs the H2 chapter heading by combining the `id` and `title` with a "–" separator.
2.  Adds a version line that includes a static version (`v1.0`) and a dynamic timestamp from `formatTimestamp`.
3.  Adds a "Status: active" line to indicate the requirement is current.
4.  Adds the H3 "Description" subheading.
5.  Appends the provided `description` content.
6.  If the `extra` string is provided, it is trimmed and appended at the end, separated by newlines, to allow for custom sections like "Traceability" or "Quality Envelope".
7.  All parts are joined into a single, trimmed markdown string.

### Output
-   (string): A complete, well-formed markdown string for the new chapter.
